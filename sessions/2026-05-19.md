## What broke / what I learned

- **`/nas` is not mounted on this host.** It's an fstab CIFS share with credentials under `/home/lab-admin/.smbcredentials` — needs lab-admin to mount it for our user. Worked around by using `/mnt/Data` (1.7 TB free). Lab-admin has since OK'd staying on `/mnt/Data` indefinitely.
- **Hit FOUR failed smoke runs before v4 succeeded.** Walked the failure chain:
  1. `smoke_v0`: stale `from nsvqa.nsvs.model_checker.frame_validator import VideoAutomaton, PropertyChecker, FramesofInterest` in `run_one()` — those symbols don't live there. Removed the dead import.
  2. `smoke_v1`: `OPENAI_API_KEY not set` — the `~/.env` file isn't auto-loaded by the OpenAI Python client. Added a `--env-file` flag and a small dotenv loader to the driver. Also: `~/.env` had no trailing newline so the naive shell `while read` missed it; my Python loader handles this fine.
  3. `smoke_v2`: NSVS OOMs on every entry — 23.45 GB / 23.55 GB used on GPU 0 with InternVL2-8B. Cause is upstream `nsvqa/nsvs/vlm/internvl.py`: `assign_device_map`/`split_model` use device-map keys (`language_model.model.embed_tokens`, `language_model.lm_head`) that don't match the actual InternVL2 module names. HF warns "device_map keys do not match any submodules", those layers stay on CPU after load, then `InternVL.move_tensors_to_gpu` shoves them all onto GPU 0. Net result: even with `--multi-gpus --gpus 0,1`, GPU 0 takes the full ~16 GB of weights + activations.
  4. `smoke_v3`: same OOM with `--multi-gpus --gpus 0,1` (confirmed the device-map keys are the real bug).
  5. `smoke_v4`: switched to InternVL2-2B (~4 GB) — fits cleanly, pipeline runs.
- **Found a real bug in `puls.process_specification`:** the substitution `if specification.count(original) == 1` skips any proposition that appears more than once in the TL spec. TimeLogic's "X before Y which in turn occurs before Z" pattern puts Y twice, so Y is left as an unquoted literal and NSVS later does `frame.object_of_interest['person moving']` → `KeyError`. Patched to `>= 1`. This single fix should recover ~125 val entries (~6%) that use the "in turn occurs" phrasing.
- **PULS spec quality on agqa "X always after Y" questions is poor.** Example: "What did the person do always after laughing at a television?" → spec `! "person_laughs_at_television"` (just the negation). The few-shot examples in `puls/prompts.py` are LongVideoBench-flavored ("girl turns on computer", "child kicks ball"); they miss TimeLogic's "person doing X always after person doing Y" verb construction. Adding 4–6 TimeLogic-style few-shots is the obvious next tuning pass (lever D — starts in session 5).
- **Upstream `evaluate.py` cannot have been recently tested:** it passes `"OpenGVLab/InternVL2_5-8B"` into `InternVL`, which re-prepends `OpenGVLab/` *and* looks up the full string in a dict that only has the short names — would `KeyError` on first run. Our driver works around it by passing the short name.
- **Architecture lesson (session 4):** spec docs under a separate folder on the server (`TimeLogic-Specs/`) are invisible to Cursor's agent context. Moving them into `.cursor/rules/` inside the fork means every agent session auto-loads them; using git as the sync layer means laptop-Claude and server-Cursor agree on what they're looking at. Session logs stay out of `.cursor/rules/` because they're append-only and would otherwise inflate the system prompt by ~100 lines per day.
