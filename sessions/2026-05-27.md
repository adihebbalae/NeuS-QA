## Strategic note — diag3 dropped; carryover lesson

Diag3 flip audit (9 rows hand-reviewed) showed GPT-5.2 reverts to language priors regardless of video duration. Reasoning summaries are direct evidence — e.g., Q65 (240.4s video): *"Considering lemonade preparation… I'm thinking about the steps for making pink lemonade"* (recipe knowledge, not visual). Q9 (2.1s): *"It feels intuitive that the answer is likely 'yes'"* (no grounding).

The pattern is not duration-dependent. Diag3 is dropped as a methodology lever (already reflected in `next-days-plan.md`), but **GPT-5.2 is still Sub7's VQA**, so the same failure mode is in flight right now. Carryover hypothesis: a visual-grounding prompt prefix (force one-sentence description of crop before the answer letter) could counter this. Prototype on the rerun rows (small N), don't touch live Sub7.

## Parallel-safe work while Sub7 VQA runs

Sub7 VQA is API-only; GPUs and the postprocess pipeline are free. Three threads available:

### Thread A — Sub7 NSVS rerun for failed rows (`scripts/run_sub7_rerun_failed_nsvs.sh`)

**Safe to run NOW in parallel with Sub7 VQA.** File-boundary analysis:

| Stage | Reads | Writes |
|---|---|---|
| Sub7 VQA (in flight) | `postprocess/postprocess_entries.json` (loaded into memory at startup) | same file at end |
| NSVS rerun (this thread) | `shard_*/diag.json`, `merged/entries.json` | `nsvs_rerun/shard_*/entries.json`, then merges into `merged/entries.json` |

VQA never re-reads `merged/entries.json` after startup, so the merge can't disturb it. **Race-free.**

Follow-up sequencing (must wait until Sub7 VQA completes):
1. Re-postprocess (ffmpeg crop) on rerun qids → **side file** (`postprocess/postprocess_entries_rerun.json`), not the live one.
2. Re-run VQA on rerun qids → side submission (`sub7_rerun_vqa/submission_partial.json`).
3. Build final Sub7 submission as union: main VQA rows ∪ rerun VQA rows (rerun overrides).

### Thread B — Sub7-vs-Sub1 stratification scaffold

Write the analysis script now, against `submission_sub1.json` + a placeholder for `submission_sub7.json`, so it's a one-command run the moment Sub7 lands. Strata to pre-bake:

- **Duration buckets:** `<2s` / `2–10s` / `10–60s` / `60–180s` / `>180s`. This is the routing-signal question (does NSVS hurt on short videos?). 50.58% of val is <30s, median 3.16s — there's real mass in the short bucket.
- **Operator family:** Boolean / MCQ / temporal-marker (until/since/while) / always-quantified.
- **Source:** agqa / star / ct / bf.
- **FOI cleanliness:** non-`-1` FOI vs `-1` FOI (NSVS fired vs bypassed).

Output: per-bucket Sub7 vs Sub1 accuracy delta. If `<2s` shows Sub1 ≥ Sub7, that's the routing evidence for a duration-gated Sub8. If `>10s` shows Sub7 ≫ Sub1, NSVS is doing real work on the bucket it's meant for.

Pair with the per-category breakdown already in `postprocess/per_category_breakdown.py` — same row keys, just a different stratification.

### Thread C — Visual-grounding VQA prompt prototype (rerun rows only)

Single-axis change to `answer_one()` in `nsvqa/vqa/answer_timelogic.py`: force the model to produce one short visual description sentence before the answer letter. Pseudocode:

```
You will see frames from a video crop. Before answering, in ONE sentence,
describe what is actually visible in the frames. Then on a new line, give
only the final answer (A/B/C/D or Yes/No).
```

Why this and not something else: the audit transcripts show the failure isn't "wrong reasoning" — it's "no reasoning from the frames at all." Forcing a visual-description token sequence anchors the next token (the answer letter) on visual content rather than recipe priors.

Why prototype on rerun rows only: Sub7 VQA is in flight on the standard prompt; don't fork that experiment. Rerun rows are a small naturally-segregated test set. If lift is real (look for: drop in third-answer flips, reasoning summaries that reference visible objects), promote to Sub8.

**Note: not a fix for `<2s` videos** — those are time-warped and don't have meaningful temporal content to ground on either way. Routing is the lever for short videos; grounding-prompt is the lever for long videos where GPT-5.2 is currently ignoring the crop.

## Decision tree once Sub7 lands

1. Read Sub7 EvalAI test score.
2. Run Thread B stratification: Sub7 vs Sub1 by duration × operator × source.
3. **If `<2s` bucket shows Sub7 ≤ Sub1:** Sub8 = duration-gated routing (Sub1 for `<2s`, Sub7 for everything else). Pure post-process, no new compute.
4. **If `<2s` bucket shows Sub7 > Sub1:** routing hypothesis is dead; pivot to grounding-prompt or PULS v2 lever.
5. **Either way:** apply the visual-grounding prompt to rerun rows and look at the deltas before deciding whether to bake it into Sub8.
